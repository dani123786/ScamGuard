## 🚀 Quick Start - 3 Steps to Deploy

### Step 1: Clean Up
```bash
python cleanup.py
```
This will automatically delete:
- Duplicate `ScamGuard` folder
- Extra documentation files
- IDE settings (.vscode)
- Test reports

### Step 2: Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit - ScamGuard v1.0"
git remote add origin https://github.com/YOUR_USERNAME/scamguard.git
git push -u origin main
```

### Step 3: Deploy on Vercel
1. Go to [vercel.com](https://vercel.com)
2. Sign in with GitHub
3. Click "New Project"
4. Import your repository
5. Click "Deploy"
6. Done! 🎉

## 🔧 Configuration Files Explained

### .gitignore
Prevents these from being committed:
- Python cache files (`__pycache__`)
- Virtual environments (`venv/`)
- IDE settings (`.vscode/`)
- User reports (`reports/*.txt`)
- Environment variables (`.env`)

### vercel.json
Tells Vercel:
- Use Python runtime
- Entry point is `app.py`
- Route all requests to Flask app

### requirements.txt
Lists Python dependencies:
- Flask (web framework)
- Other required packages
